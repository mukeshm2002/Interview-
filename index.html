‚ú® Core Java (OOPs, Collections, Exception Handling)

Class & Object :

  • In Java, a **Class** is like a blueprint or template that defines how an object should be ‚Äî it contains variables and methods.
  • An **Object** is an actual instance of a class ‚Äî it‚Äôs created from that blueprint and has real values.

OOPS 4 Pillars :

  • **Encapsulation**: Wrapping (or combining) data (variables) and methods (functions) that operate on that data into a single unit (class).
  • **Inheritance**: A class (child) can inherit the properties and methods of another class (parent).
  • **Polymorphism**: The same method can behave differently based on the object calling it. Types: Compile-time (**Method Overloading**), Runtime (**Method Overriding**)
  • **Abstraction**: Showing only the important details and hiding the implementation.

Method Overloading and Method Overriding :

  • **Method Overloading** happens within the same class, when multiple methods have the same name but different parameters (different type, number, or order). It‚Äôs a **compile-time polymorphism**.
  • **Method Overriding** happens in different classes ‚Äî a child class provides a new version of a method that already exists in its parent class. It‚Äôs a **runtime polymorphism**.

Abstract class vs Interface :

An **abstract class** is used when we want to share common behavior among subclasses, and it can have both abstract and normal methods.

An **interface** defines a contract that classes must follow ‚Äî it supports multiple inheritance and achieves full abstraction.

> *Usecase*: In my Bus Booking Project, I used **interfaces** like `BookingService` and `PaymentService` to define structure, and **abstract classes** like `User` for shared properties between Admin and Customer.

Exception Handling (and Custom Exceptions):

In Java, **Exception Handling** is a way to manage runtime errors so that the application doesn‚Äôt crash and can handle unexpected situations smoothly.

> *Usecase*: In my Online Bus Booking Project, I used **custom exceptions** to handle specific error cases clearly and user-friendly. For example: `NoRouteFoundException`, `BusNotFoundException`, `InsufficientSeatException`, and `SeatAlreadyBookedException`. These custom exceptions made my application more user-friendly, maintainable, and professional.

Throw and Throws :

  • The **throw** keyword is used to manually throw an exception inside a method or block of code. It‚Äôs followed by an object of an exception class (like `new Exception()`).
  • The **throws** keyword is used in a method declaration to tell the compiler that this method might throw an exception, and it should be handled by the calling method.

String, StringBuilder and StringBuffer :

In Java, **String** is **immutable** ‚Äî once created, it can‚Äôt be changed.
**StringBuilder** and **StringBuffer** are **mutable**, meaning they can be modified.
The difference is ‚Äî **StringBuilder** is faster but **not thread-safe**, while **StringBuffer** is **thread-safe** but a bit slower.
> *Usecase*: I usually prefer **StringBuilder** when working on single-threaded applications for better performance.

üå± Spring Boot & Spring Data JPA

Spring Boot :

Spring Boot is a framework built on top of the Spring Framework. Its main goal is to make it easier and faster to start and develop Spring applications.

The main idea is **‚ÄúConvention over Configuration‚Äù** (‡Æ§‡Ææ‡Æ©‡Ææ‡Æï‡Æµ‡Øá ‡Æï‡Æü‡Øç‡Æü‡ÆÆ‡Øà‡Æ§‡Øç‡Æ§‡Æ≤‡Øç), meaning it uses smart defaults so we don‚Äôt have to set up everything ourselves. Another big advantage is **embedded servers** (like Tomcat), which let us run the app directly as a single JAR file.

Spring MVC Architecture (Model-View-Controller) :

Spring MVC helps to separate the **logic (Model)**, **design (View)**, and **user input (Controller)** of an application.

Controller ‚Üí calls the Model (Service/Repo) ‚Üí Model handles business logic/data ‚Üí Controller passes data to the View.

> *Usecase*: In my Employee Management System, the **Controllers** handled API requests, the **Services** managed the business logic, and the **Repositories** connected with the database.

Rest API :

A **RESTful API** (Representational State Transfer) is used to build communication between the client and server using standard HTTP methods.
  • **GET** ‚Üí to fetch data
  • **POST** ‚Üí to add new data
  • **PUT** ‚Üí to update existing data
  • **DELETE** ‚Üí to remove data
We use the `@RestController` annotation and `@GetMapping`, `@PostMapping`, etc., to map URLs to methods. The methods usually return **JSON** data.

Database Connectivity (Spring Data JPA) :

Connecting Spring Boot to a database is very easy with **Spring Data JPA**.
  1. Add `spring-boot-starter-data-jpa` dependency.
  2. Set up database URL, username, and password in `application.properties`.
  3. Create data models as **JPA entities** (e.g., `@Entity` Employee class).
  4. Make **repository interfaces** by extending `JpaRepository`, which automatically gives all basic **CRUD** operations.

Spring Security :

Spring Security is a powerful framework that provides **authentication** (verifying who the user is) and **authorization** (checking what the user is allowed to do).

Adding `spring-boot-starter-security` sets up basic security. It can be customized by defining user **roles** and securing APIs with **JWT** (JSON Web Token) for stateless authentication.

Spring Thymeleaf (View Layer) :

Spring **Thymeleaf** is a **server-side template engine** used to create dynamic web pages.

The Controller sends data to the Thymeleaf template, where we use expressions like `${employee.name}` to display that data inside HTML. Thymeleaf processes these placeholders and sends the final HTML page to the browser. This keeps the design separate from business logic.

Test Cases Integration (Mockito) :

I used **Mockito** with **JUnit** for writing **unit test cases** in Spring Boot.
Mockito helps me test the logic of a **Service** class by creating **mock** dependencies (like a Repository) using `@Mock` and `@InjectMocks`. This allows me to focus only on testing the business logic without hitting the real database, making testing faster and more reliable.
üé® Frontend (HTML/CSS/JS)

Semantic HTML :

Semantic HTML elements provide **structure and meaning** to web content, making it more accessible and understandable for browsers and developers. Examples: `<header>`, `<footer>`, `<article>`, `<section>`, `<nav>`.

CSS Box Model :

The CSS box model describes every element as a box with **content**, **padding**, **border**, and **margin**. It's crucial for designing layouts.

Key CSS Properties :

Commonly used properties include `margin` and `padding` for spacing, `color` and `font-size` for typography, `display` and `position` for layout, and `border` for visual separation.

DOM Manipulation :

**DOM manipulation** is using JavaScript to dynamically change the **content**, **structure**, or **style** of a webpage, making it interactive.

Event Loop in JavaScript :

The **event loop** is a mechanism that handles **asynchronous code execution** and ensures that events are processed when the main thread (Call Stack) is available (Non-blocking I/O).
‚òÅÔ∏è Git & GitHub

**Git** is a distributed **version control system** that lets developers track changes in their code and collaborate on projects. **GitHub** is a web-based platform that provides **hosting** for Git repositories, along with features for collaboration and code review.

Key Git Commands:

  • **`git init`**: Creates a new Git repository in the current folder.
  • **`git clone`**: Creates a copy of a remote Git repository on your local machine.
  • **`git status`**: Shows the current state of your repository.
  • **`git add`**: Adds files to the staging area.
  • **`git commit`**: Creates a snapshot of your project's changes.
  • **`git push`**: Sends your committed changes from local to a remote repository.
  • **`git pull`**: Fetches and merges changes from a remote repository to your local one.
  • **`git branch`**: Manages branches (creating, listing, deleting).
  • **`git checkout`**: Switches between branches or restores files.
  • **`git merge`**: Combines the changes from one branch into another.
