1. Java (Core, OOPs, Collections & DSA)

OOPs & Core Concepts

Encapsulation: Wrapping data and methods into a single class, protecting internal state.

Abstraction: Hiding complex implementation, exposing only essential information (e.g., using service interfaces in Spring).

Inheritance: A class inheriting properties/methods from another. (Reuse code).

Polymorphism: An action working in different ways (Method Overloading/Overriding).

Overloading: Same method name, different parameters, Compile Time polymorphism.

Overriding: Subclass redefines parent method (same signature), Runtime polymorphism.

Abstract Class: Can have abstract and non-abstract methods, allows instance variables, classes extend it. (Partial implementation).

Interface: Only abstract methods (before Java 8), only constants, classes **implement** it. (Defines a contract).

  • public: Accessible anywhere.
  • private: Accessible only within the class.
  • protected: Accessible within the same package and by subclasses.
  • default (package-private): Accessible only within the same package.
Collections & Java 8+ Features

It's a unified architecture for handling groups of objects. Core interfaces are List (ordered, allows duplicates), Set (unordered, no duplicates), and Map (Key-Value pairs).

PROJECT FOCUS: I use ArrayList for fetching all entity records and HashMap for fast lookups based on unique IDs (like an Employee ID in the EMS).

ArrayList: Uses a dynamic array. Fast searching/access . Slow insertion/deletion in the middle .

  • PROJECT USE: Used in EMS for displaying a list of records (reads are frequent).

LinkedList: Uses a doubly-linked list. Slow searching . Fast insertion/deletion.

  • PROJECT USE: Used in Patient Mgmt for sequence-based data where new items are constantly added or removed from the ends.

Introduced in Java 8. Provides a functional, declarative way to process collections via chaining operations like .filter(), .map(), and .collect(). It avoids modifying the source collection (non-mutating).

Separates concerns into Model (data), View (presentation, e.g., Thymeleaf), and Controller (handles requests). Spring Boot uses the Controller - Service - Repository pattern, which is an extension of MVC.

Simplifies database access. I extend the `JpaRepository` interface, which automatically provides CRUD (Create, Read, Update, Delete) operations. I write very little, boilerplate-free data access code. I can also define custom finder methods like `findByRouteName()`.

Stateless web services using standard HTTP methods to manipulate resources (Entities/Data). I use `@RestController` in Spring to expose endpoints:

  • GET: Retrieve resources (e.g., `/employees`).
  • POST: Create a new resource (e.g., `/employees`).
  • PUT/DELETE: Update or delete a specific resource.
Security, Testing & Deployment

Handles Authentication (who you are) and Authorization (what you can do). PROJECT FOCUS: I used it in the Bus Reservation System to enforce different access for Admins (to add routes) and Users (to book tickets).

3. Frontend (HTML, CSS, JavaScript) - No Frameworks

HTML & CSS Core

Elements that clearly describe their meaning, like <header>, <nav>, <main>, and <footer>. They improve SEO and Accessibility (making the page structure clear for screen readers).

It defines how an element's total space is calculated, consisting of four parts, from inside out:

  1. Content: The actual element size (width/height).
  2. Padding: Space between the Content and the Border.
  3. Border: The line surrounding the element and its padding.
  4. Margin: Clear space outside the Border, separating the element from others.

I use `box-sizing: border-box;` to make width/height include padding and border, simplifying layout calculations.

Flexbox: A 1-dimensional layout system (rows OR columns). Great for aligning items inside a component, like navigation links or form elements.

CSS Grid: A 2-dimensional layout system (rows AND columns simultaneously). Used for creating the overall page layout structure (e.g., header, sidebar, main content).

Media Queries are a CSS feature that allows applying styles only when a certain condition is met, most commonly the device's screen size. This is essential for Responsive Design.

PROJECT USE: I used them in **EMS** to hide or rearrange the sidebar navigation when the screen width drops below a tablet size (`@media (max-width: 768px)`), providing a mobile-friendly experience.

JavaScript & DOM Manipulation

The Document Object Model is a programming interface for web documents. It represents the page as a tree of nodes (elements, text, attributes) that JavaScript can access and manipulate to dynamically change content, structure, and styles.

These are the two ways events propagate through the DOM tree:

  • Capturing (Top-Down):The event starts at the `window` and travels down to the target element.
  • Bubbling (Bottom-Up):The event starts at the target element and travels up to the `window`. Bubbling is the default behavior in most browsers.

**Event Delegation** uses the **Bubbling** phase by placing one listener on a parent element to manage events for all its children, which is great for performance.

A closure is the ability of an inner function to access the variables of its outer function even after the outer function has finished executing.

It's used to create private variables and maintain state. In my projects, I use it when generating dynamic form elements to ensure each element retains its correct, private ID for handling data submission.

  • var:Function-scoped. Can be redeclared and updated. Hoisted (accessible before declaration). Avoided in modern JS.
  • let:Block-scoped (inside `{}`). Cannot be redeclared but can be updated.
  • const:Block-scoped. Cannot be redeclared or updated (must be initialized). Used for values that should not change.

I prioritize `const` by default, then `let` only if a variable needs reassignment, and I avoid `var` entirely for clean code.

Asynchronous Operations (The Core of Frontend)

AJAX (Asynchronous JavaScript and XML)** is the technique of making requests to a server without reloading the page. Fetch is the modern standard for this.

PROJECT USE: In the Bus Reservation System, I use `fetch()` to call my Spring Boot REST APIs (e.g., `fetch('/api/routes')`). It returns a Promise, which I handle using .then() to process the JSON data and update the DOM.

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. It acts as a placeholder for a value that will become available in the future.

It has three states: Pending (initial), Fulfilled (successful completion), or Rejected (failed). It helps manage asynchronous flow cleanly, avoiding Callback Hell.

Both are Web Storage APIs for storing key-value pairs locally in the browser.

  • localStorage: Data persists even after the browser is closed. PROJECT USE: Storing user preferences or theme settings.
  • sessionStorage: Data is cleared when the browser tab/window is closed. PROJECT USE: Storing temporary data like a multi-step form's current progress or a logged-in user's temporary state.
