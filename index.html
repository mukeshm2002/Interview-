2. What is a Java Full Stack Developer?

1‑minute answer
A Java Full‑Stack Developer works on both client and server sides. On the back end we use Java and frameworks like Spring Boot to implement business logic, security and data persistence (MySQL). On the front end we build UIs with HTML, CSS and JavaScript — or server rendering with Thymeleaf — and increasingly use React for dynamic interfaces. Full‑stack means owning the data flow end‑to‑end.

3. What is Spring Boot?

1‑minute answer
Spring Boot is a convention‑based framework built on Spring that lets you create production‑ready Java applications with minimal configuration. It provides auto‑configuration, embedded servers (Tomcat), starter dependencies and features like Actuator for monitoring. It reduces boilerplate so you can focus on business logic and build REST APIs, microservices and web apps quickly.

4. What is Front‑End Development?

1‑minute answer
Front‑end development builds the part of the application users interact with in their browser. It uses HTML for structure, CSS for styling and JavaScript for behaviour. I have built server‑rendered pages with Thymeleaf and plan to learn React to create more dynamic single‑page applications that communicate with my Spring Boot APIs.

5. Class and Objects (with example)

1‑minute answer
A class is a blueprint that defines attributes and behavior. An object is an instance of that class. Example: class Car { String color; void start() { } } — Car myCar = new Car(); myCar.color = "Red"; myCar.start(); The class defines structure; the object represents a real entity.

6. Method Overloading vs Overriding

1‑minute answer
Overloading is having methods with the same name but different parameters in the same class — compile‑time polymorphism. Overriding is redefining a parent class method in a child class with the same signature — runtime polymorphism. Use overloading for convenience, overriding to change behavior in subclasses.

7. Exception Handling (with Bus Booking custom exceptions)

1‑minute answer
Exception handling prevents crashes by catching and managing errors. In my Bus Booking project I created custom exceptions: NoRouteFoundException, BusNotFoundException, InsufficientSeatException and SeatAlreadyBookedException. Throwing and catching these gives clear user messages and keeps the app stable instead of returning generic errors.

8. throw vs throws

1‑minute answer
Use `throw` to explicitly raise an exception object inside a method. Use `throws` in a method signature to declare that the method may pass exceptions to its caller. `throw` creates the exception; `throws` informs callers they must handle or rethrow it.

9. String vs StringBuilder vs StringBuffer

1‑minute answer
`String` is immutable — changes create new objects. `StringBuilder` is mutable and fast but not thread‑safe, ideal for single‑threaded scenarios. `StringBuffer` is mutable and synchronized (thread‑safe) but slightly slower. Choose based on concurrency and performance needs.

10. Java Features (Java 8 → 25)

1‑minute answer
Java 8 added lambdas, streams and the new date API — a major release. Later versions added modules (9), `var` (10), records, text blocks, pattern matching, sealed classes, virtual threads, and performance improvements. Java 17 and 21 are LTS releases focusing on virtual threads and records. Keep answers concise and mention a few features you use.

11. Abstract Classes and Interfaces

1‑minute answer
Abstract classes can have implemented methods and fields and are used when classes share behavior. Interfaces define contracts (methods) that classes implement and support multiple inheritance. Use abstract classes for shared base functionality and interfaces for loose contracts and multiple implementations.

12. Java Collections Framework

1‑minute answer
The Collections Framework offers List, Set, Map and Queue interfaces and implementations like ArrayList, LinkedList, HashSet and HashMap. It provides utilities for searching, sorting and collection manipulation. I use collections for storing and processing bus lists, bookings and mappings in my projects.

13. Stream API & Lambda Expressions

1‑minute answer
Lambdas provide concise function expressions; Stream API enables declarative data processing like filter, map and collect. In the Bus Booking project I used streams to filter available buses, sort by departure time and map results for the frontend — making code shorter and expressive.

14. Arrays (with work experience)

1‑minute answer
Arrays store fixed‑size homogeneous data. I used arrays for simple seat maps and temporary route lists during development, e.g., boolean[] seats to mark booked seats. After prototyping, I moved data to database tables and Collections for scalability.

15. Printing patterns (programming)

1‑minute answer
Pattern printing tests nested loops and logic. Explain row/column loops, and demonstrate a simple star or number pattern. It shows basic control flow understanding and is a good warm‑up question in interviews.

16. What you know about DSA and how to use in your projects

1‑minute answer
DSA covers arrays, lists, stacks, queues, trees, graphs and algorithms like sorting, searching and traversal. I apply DSA to choose appropriate structures — e.g., HashMap for fast lookups, ArrayList for ordered reads — improving performance in search, booking lookups and data processing.

17. Design Patterns and how to use

1‑minute answer
Design patterns are reusable solutions to common design problems. I used Singleton for a single DB connection, Factory to create role‑based objects, and Strategy/Observer where behavior needed to be interchangeable. Patterns improve code readability and maintainability.

18. RESTful APIs using projects

1‑minute answer
RESTful APIs use HTTP verbs (GET, POST, PUT, DELETE) for CRUD. In my projects I built controllers for buses, bookings and users in Spring Boot, returned JSON responses, and secured endpoints with JWT. APIs made the frontend/backed integration simple and modular.

19. Spring Security

1‑minute answer
Spring Security provides authentication and authorization. I implemented JWT for stateless auth: users log in, receive a token and include it with requests. Role‑based access (Admin/User) restricts APIs and BCrypt encrypts passwords for safety.

20. Spring Boot Database Connectivity

1‑minute answer
Spring Boot connects to databases using Spring Data JPA and auto‑configuration. Add JPA and connector dependencies, set `spring.datasource` properties, define entities and JpaRepository interfaces. Spring handles sessions and CRUD via repositories — I used this with MySQL in my apps.

21. Git and GitHub with most used commands

22. Spring MVC Architecture with project experience

1‑minute answer
Spring MVC follows Model, View, Controller. DispatcherServlet routes requests to controllers, controllers call services and repositories, models carry data and views (Thymeleaf) render HTML. I used this flow in Bus Booking to serve search and booking pages.

23. Abstract Classes and Interfaces

1‑minute answer
Abstract classes allow shared implementation and abstract methods; interfaces declare contracts and can be implemented by many classes. Use abstract base classes for common behavior and interfaces for service contracts (e.g., BookingService interface implemented by BookingServiceImpl).

24. Java Collections Framework

34. DispatcherServlet

1‑minute answer
DispatcherServlet is the front controller in Spring MVC. It receives requests, finds the appropriate controller via HandlerMapping, executes controller logic, and uses a view resolver to render responses. In Spring Boot it is configured automatically and routes endpoints to controllers in my projects.
